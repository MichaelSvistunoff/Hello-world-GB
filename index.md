# Использовани GIT в контроле версий [Git documentation]

## Базовый ситаксис [Basic syntax]

### Заголовки [Headings]

Чтобы создать заголовок, добавьте знаки цифр (#) перед словом или фразой. Количество используемых вами цифровых знаков должно соответствовать уровню заголовка. Например, чтобы создать заголовок третьего уровня, используйте три цифровых знака (например, ### My Header)

# Заголовок первого уровня
## Заголовок второго уровня 
### Заголовок третьего уровня
#### Заголовок четвертого уровня
##### Заголовок пятого уровня
###### Заголовок шестого уровня  

Либо в строке под текстом добавьте любое количество ==символов для уровня заголовка 1 или --символов для уровня заголовка 2. Например:

Заголовок первого уровня
===

Заголовок второго уровня 
---

### Параграф [Paragraphs]

Чтобы создать абзацы, используйте пустую строку для разделения одной или нескольких строк текста. Например:

Мне очень нравится использовать Markdown.

Думаю, теперь я буду использовать его для форматирования всех своих документов.

### Разрывы строк [Line Breaks]

Чтобы создать разрыв строки или новую строку, завершите строку двумя или более пробелами, а затем введите Enter. Например:

Это первая строка.  
А это вторая строчка.

#### Жирный [Bold]

Чтобы выделить текст жирным шрифтом, добавьте две звездочки или символы подчеркивания до и после слова или фразы. Чтобы выделить середину слова жирным шрифтом, добавьте две звездочки без пробелов вокруг букв

Первый вариант выделения текста **жиирным**  
Второй вариант выделения текста __жирным__

### Курсив [Italic]

Чтобы выделить текст курсивом, добавьте одну звездочку или подчеркивание до и после слова или фразы. Чтобы выделить середину слова курсивом, добавьте одну звездочку без пробелов вокруг букв

Первый вариант выделения текста *курсивом*  
Второй вариант выделения текста _курсивом_

### Комбинация жирного и курсива [Bold and Italic]

Чтобы выделить текст жирным шрифтом и курсивом одновременно, добавьте три звездочки или знака подчеркивания до и после слова или фразы. Чтобы выделить середину слова полужирным шрифтом и выделить его курсивом, добавьте три звездочки без пробелов вокруг букв

Первый вариант выделения текста ***жирным курсивом***  
Второй вариант выделения текста ___жирным курсивом___  
Третий вариант выделения текста __*жирным курсивом*__  
Четвертый вариант выделения текста **_жирным курсивом_**

### Цитаты [Blockquotes]

Чтобы создать цитату, добавьте > перед абзацем

> ## Надо любить жизнь больше, чем смысл жизни.
>
> *Федор Достоевский*

### Вложенные цитаты [Nested Blockquotes]

Цитаты могут быть вложенными. Добавьте >>перед абзацем, который вы хотите вложить

> Мы работаем, чтобы иметь свободное время, и воюем, чтобы жить мирно.
>
>> Два года человек учится говорить, а потом всю оставшуюся жизнь — молчать.
>
> *Аристотель*

### Списки [Lists]

Вы можете организовать элементы в упорядоченные и неупорядоченные списки

#### Упорядоченные списки [Ordered Lists]

Чтобы создать упорядоченный список, добавьте позиции с номерами, за которыми следуют точки. Номера не обязательно должны быть в порядке номеров, но список должен начинаться с единицы

1. Элемент 1
2. Элемент 2
3. Элемент 3
4. Элемент 4  
    1. Элемент 4.1  
    2. Элемент 4.2
    3. Элемент 4.3

#### Неупорядоченные списки [Unordered Lists]

Чтобы создать неупорядоченный список, добавьте дефисы ( -), звездочки ( *) или знаки плюс ( +) перед элементами строки. Сделайте отступ для одного или нескольких элементов, чтобы создать вложенный список

- Элемент 1
- Элемент 2

* Элемент 1
* Элемент 2

+ Элемент 1
+ Элемент 2

- Элемент 1
- Элемент 2
- Элемент 3
    - Элемент 3.1
    - Элемент 3.2

Чтобы быстро превратить URL-адрес или адрес электронной почты в ссылку, заключите их в угловые скобки

<fake@example.com>

### Код [Code]

Чтобы обозначить слово или фразу как код, заключите его в обратные кавычки (`)

В командной строке введите `нано`.

### Экранирование обратных кавычек [Escaping Backticks]

Если слово или фраза, которые вы хотите обозначить как код, содержат одну или несколько обратных кавычек, вы можете избежать их, заключив слово или фразу в двойные обратные кавычки ( ``)

``Use `code` in your Markdown file.``

### Разделители блоков [Horizontal Rules]

Чтобы создать горизонтальную линейку, используйте три или более звездочек ( ***), дефисов ( ---) или символов подчеркивания ( ___) в отдельной строке.

---

***

___

## Стандартные команды [Main commands]

### Подготовка репозитория [repository's preparing]

Для создание репозитория необходимо выполнить команду: 
<pre>git init</pre>  
в папке с репозиторием и у Вас создаться репозиторий (появится скрытая папка <mark>.git</mark>)

### Создание коммитов [commit creation]

#### Подготовка к коммиту [Git add]

***Для добавления измений*** в коммит используется команда `git add`. Чтобы использовать команду git add напишите <pre>`git` add <file_name.extension></pre>

\* Чтобы не печатать полностью название файла, нужно ввести первые две буквы его названия и нажать **Tab**

#### Создание коммитов [commit creation]

Для того, чтобы создать коммит(сохранение) необходимо выполнить команду `git commit`. Выполняется она так: <pre>`git` commit -m "<commit_message></pre> Все файлы для коммита должны быть **ДОБАВЛЕНЫ** и сообщение к коммиту писать **ОБЯЗАТЕЛЬНО**

### Просмотр состояния репозитория [view repository's status]

Для того, чтобы посмотреть состояние репозитория используется команда `git status`. Для этого необходимо в папке с репозиторием написать <pre>`git` status</pre> и Вы увидите были ли измения в файлах, или их не было

### Перемещение между сохранениями [choosing save]

Для того, чтобы перемещаться между коммитами, используется команда `git checkout`. Используется она в папке с пепозиторием следующим образом: <pre>`git` checkout <commit_number></pre>

\* Можно скопировать 4 первые символа нужного коммита

### Журнал изменений [changes log]

Для того, чтобы посмтреть все сделанные изменения в репозитории, используется команда `git log`. Для этого достаточно выполнить команду <pre>`git` log</pre> в папке с репозиторием

## Ветки в Git [branches in Git]

### Создание ветки [branch's creation]

Для того, чтобы создать ветку, используется команда `git branch`. Делается это следующим образом в папке с репозиторием: 
<pre>`git` branch <new_branch_name></pre>

### Слияние веток [branch's merge]

Для того чтобы дабавить ветку в текущую ветку используется команда <pre>`git merge <name_branch>`</pre>

### Удаление веток [branch's delete]

Для удаления ветки ввести команду <pre>`git` branch -d 'name branch'</pre>
